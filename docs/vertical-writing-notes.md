# 縦書きテーマの注意事項

`Japanese Novel Vertical` と `Japanese Novel Vertical Dark` は、CSS の
`writing-mode: vertical-rl` を利用した実験的な小説用テーマです。縦書きの
閲覧と A4 PDF 出力を主な用途としています。

縦書きは本文エディタ `#write` のみに適用します。サイドバー、ファイル一覧、
アウトライン、メニュー、タイトルバーなどTyporaのUIは横書きのままです。

## Typora 上の制約

Typora の縦書き対応は限定的です。テーマで本文を縦組みにすることは
できますが、横スクロールの位置、キャレットの移動、矢印キー、文頭への
移動などは、横書き時と同じ操作感にならない場合があります。長文の編集は
横書きテーマで行い、縦書きテーマは閲覧や PDF 出力時に選択する運用が
安定しています。

ソースコードモードは、CodeMirror の操作性を優先して横書きに戻します。

HTMLで補助クラスを記述した段落は、カーソルがある間だけTyporaのHTML編集表示が
横書きになる場合があります。カーソルを外した閲覧状態とPDF出力では縦書きに
戻ります。これはTyporaの編集UIによる制約です。

`<p class="dialogue">...</p>` のようなブロックHTMLは、閲覧中も横方向へ大きな
編集領域を確保することがあります。補助クラスは、独立した行に
`<span class="dialogue">...</span>` と記述する方法を推奨します。

YAMLフロントマターも横書き編集領域が大きいため、縦書きテーマの画面では非表示に
します。値の編集は横書きのソースコードモードで行ってください。PDF出力用の設定
としては引き続き利用できます。

画面表示では、短い段落のたびに縦列の残り全体が空白になることを避けるため、
隣接する本文段落を同じ縦列へ続けて流します。段落境界は1字分の空きで示します。
見出し、引用、画像、表などは独立したブロックです。PDF出力と印刷では、
一般的な縦組と同じく各段落を新しい縦列から始めます。

## 対応している表現

- 通常の日本語本文、句読点、括弧
- HTML の `<ruby>` と `<rt>` によるルビ
- Markdown の強調 `*...*` による縦書き用の圏点
- `<span class="tcy">12</span>` による縦中横
- 引用、区切り線、章見出し
- 本文と重ならない独立配置のMarkdown画像
- セル内を縦組みにしたMarkdown表
- `<h2 class="chapter">...</h2>`（または `chapter-title`）による印刷時の章改ページ

縦中横はすべての数字に自動適用されるわけではありません。2 桁程度の数字や
短い英数字のみを `.tcy` で指定してください。

## 画像と表

Markdown画像は、画像を含む段落全体を独立したブロックとして扱い、画像の物理的な
横幅を本文フロー内に確保します。これにより、幅の広い画像が隣接する本文列へ重なる
ことを防ぎます。画像は画面と印刷領域に収まるよう縮小します。

Markdown表は、表全体と各セルに `vertical-rl` を適用します。Markdownの先頭行は
右端の縦列になり、後続行は左へ並びます。Typoraの行・列追加などの表編集UIは
横書き前提のため、操作部分だけ横書きのままです。複雑な表や列数の多い表では、
横書きテーマまたは画像化した表の利用を推奨します。

## 横組みに戻す要素

コードブロック、数式、ダイアグラムなどは、ライブラリやTyporaの編集UIが横組みを
前提としています。これらは局所的に横書きへ戻し、スクロールまたは縮小で表示します。
長いコードや複雑な数式は、縦書きPDFでは崩れたり分割されたりする可能性があります。

## ダークテーマと印刷

Dark 版の濃色背景は画面表示にのみ使用します。PDF 出力と印刷では、ライト版と
同じく白背景・黒文字に強制します。Typora の PDF オプションで「背景のグラフィック」
に相当する設定を変更しても、テーマ側では白紙での出力を前提とします。

## PDF の確認

縦書き PDF は `vertical-rl` により、各ページ内の行が右から左へ進むように
構成しています。ただし、PDF エンジンや Typora のバージョンによりページ分割の
結果が変わることがあります。公開や入稿の前に、ルビ、縦中横、改ページ、画像、
最終ページの欠落がないことを、出力した PDF で確認してください。
